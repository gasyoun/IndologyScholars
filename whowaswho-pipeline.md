# **Архитектура конвейера Indology Scholars: Руководство**

В этом документе описывается полный процесс извлечения данных об индологах с сайта whowaswho-indology.info, их обработки и интеграции. Мы используем полуавтоматический подход с привлечением ChatGPT Plus, чтобы избежать дополнительных затрат на API и блокировки аккаунта, а затем генерируем файлы Markdown и экспортируем события в Google Calendar.

## **Шаг 1: Сбор сырых данных (scraper.py)**

Этот этап отвечает за загрузку всех биографических данных и сохранение их в базу данных. Скрипт перебирает алфавитный указатель сайта и собирает весь текст, относящийся к каждому ученому, без потери информации.

* Создает или подключается к локальной базе данных SQLite indology\_scholars.db.  
* Проходит по всем страницам на сайте.  
* Объединяет абзацы (\<p\>), списки и другие элементы в единый блок текста для каждого ученого, ориентируясь на формат заголовков (фамилия заглавными буквами).  
* Сохраняет сырой текст в таблицу scholars.

## **Шаг 2: Подготовка данных для ChatGPT (prepare\_batches.py)**

Поскольку мы используем ChatGPT Plus вручную, этот скрипт делит сырые данные на удобные порции (батчи) и создает текстовые файлы с готовыми запросами (промптами).

* Извлекает сырые биографии из базы данных, которые еще не были обработаны.  
* Генерирует файлы в папке chatgpt\_prompts/ (например, batch\_001.txt).  
* Каждый файл содержит инструкцию для ChatGPT (системный промпт) и набор биографий, требуя вернуть данные в строгом формате JSON.

## **Шаг 3: Взаимодействие с ChatGPT (Ручной этап)**

Этот этап выполняется пользователем.

1. Откройте сгенерированный файл из chatgpt\_prompts/.  
2. Скопируйте текст и отправьте его в чат ChatGPT Plus.  
3. Скопируйте полученный JSON-ответ.  
4. Сохраните ответ в папку chatgpt\_results/ с расширением .json (например, result\_001.json).

## **Шаг 4: Импорт структурированных данных (import\_json.py)**

Скрипт читает полученные JSON-файлы и распределяет данные по реляционным таблицам в базе.

* Создает таблицы structured\_scholars, scholar\_tags и scholar\_bibliography, если они не существуют.  
* Читает все файлы .json из папки chatgpt\_results/.  
* Связывает структурированные данные с исходными записями с помощью raw\_text\_id и загружает их в базу.

## **Шаг 5: Генерация памятных дат (query.py)**

Модуль извлекает даты рождения и смерти из базы данных, вычисляет круглые даты и создает Markdown-файлы.

* Считывает структурированные данные из базы.  
* Вычисляет юбилеи (например, 50, 100 лет) для заданного периода (по умолчанию 20 лет, начиная с 2026 года).  
* Создает папку jubilees/ и сохраняет в нее Markdown-файлы (например, anniversaries\_2026.md) с YAML frontmatter, готовые для цифрового репозитория.

## **Шаг 6: Интеграция с Google Calendar (calendar\_sync.py)**

Последний этап связывает базу данных с вашим календарем, создавая ежегодные повторяющиеся события.

* Требует файл credentials.json из Google Cloud Console и аутентифицирует пользователя.  
* Находит календарь с именем "Indology" в вашем аккаунте.  
* Читает даты из базы данных. Если известны только годы, использует 30 января как резервную дату.  
* Добавляет события в Google Calendar как ежегодные (recurrence).
