# РИНЦ / eLIBRARY Manual Review Guide

> **Версия:** 1.0 · **Дата:** 2026-05-21  
> **Область применения:** проект IndologyScholars — верификация учёных в Научной электронной библиотеке (eLIBRARY.ru / РИНЦ)

---

## 1. Назначение

Этот документ описывает процедуру ручной верификации записей учёных в базе РИНЦ/eLIBRARY.ru.  
**Массовый скрейпинг не применяется.** Workflow является полностью ручным или полуавтоматическим:

1. Генератор создаёт очередь (`analytics_output/authority_review_queue.csv`) с готовыми поисковыми URL.
2. Рецензент кликает по ссылке, открывает eLIBRARY.ru и проверяет профиль вручную.
3. Найденные и подтверждённые `rinc_author_id` и `spin` вносятся в `authority_ids.json` с полями `confidence` и `checked_at`.

---

## 2. Как читать очередь проверки

Файл `analytics_output/authority_review_queue.csv` содержит следующие колонки:

| Колонка | Описание |
|---|---|
| `priority_rank` | 1 = активный учёный без каких-либо внешних ID; 4 = есть ID, но нет метаданных |
| `person_id` | ID в базе IndologyScholars (стабильный, используется в URL страниц) |
| `display_name` | Отображаемое имя (может быть с инициалами) |
| `full_name_ru` | Полное кириллическое имя если известно (Фамилия Имя Отчество) |
| `total_talks` | Количество докладов в корпусе |
| `reason` | Причина попадания в очередь |
| `rinc_search_url` | Ссылка на поиск автора в eLIBRARY.ru по фамилии/имени/отчеству |
| `openalex_search_url` | Ссылка на поиск автора в OpenAlex (по латинскому имени) |
| `review_status` | Рабочий статус: `todo` / `candidate` / `confirmed` / `ambiguous` / `not_found` |

---

## 3. Алгоритм верификации в eLIBRARY.ru

### 3.1 Открыть поисковую ссылку

Столбец `rinc_search_url` содержит предзаполненный URL вида:

```
https://elibrary.ru/authors.asp?fams=Цветкова&imas=Софья&otchs=Олеговна
```

Параметры: `fams` — фамилия, `imas` — имя, `otchs` — отчество.  
Если имя или отчество неизвестно (только инициалы), параметры будут содержать только первую букву.

### 3.2 Оценить результаты поиска

Для каждого кандидата проверьте:

| Сигнал | Что ищем |
|---|---|
| **Организация** | Совпадает ли институт с тем, что зафиксирован в корпусе конференций (ИВ РАН, СПбГУ, МГУ, ИФ РАН и т. п.)? |
| **Тематика** | Индология, востоковедение, буддология, философия, религиоведение, этнография |
| **Годы активности** | Пересекаются ли публикации РИНЦ с годами участия в Зографских/Рериховских чтениях? |
| **Соавторы** | Есть ли общие фамилии с другими участниками корпуса? |
| **Список публикаций** | Темы публикаций соответствуют тематике докладов в архиве? |

### 3.3 Правила принятия решения

- **confirmed** — организация, тематика и хронология однозначно совпадают. Нет разумного сомнения.
- **candidate** — вероятное совпадение, но нет достаточно уникальных подтверждающих сигналов. Требуется второй рецензент.
- **ambiguous** — несколько профилей с похожими именами; невозможно определить без дополнительных данных.
- **not_found** — поиск не вернул ни одного разумного кандидата.

## 4. Как вносить подтверждённые ID

После подтверждения откройте `authority_ids.json` и добавьте/обновите запись:

```json
"PERS_c52dfaa0": {
  "preferred_latin_name": "Sofia Tsvetkova",
  "rinc_author_id": "123456",
  "spin": "0000-0000",
  "confidence": "confirmed",
  "source": "manual",
  "checked_at": "2026-05-21",
  "notes": "ИВ РАН, совпадение тем и хронологии."
}
```

**Обязательные поля при добавлении любого внешнего ID:**

| Поле | Значение |
|---|---|
| `confidence` | `"confirmed"` / `"candidate"` / `"ambiguous"` |
| `source` | `"manual"` (всегда для РИНЦ) |
| `checked_at` | `"YYYY-MM-DD"` — дата проверки |

**Никогда не добавляйте** `rinc_author_id` или `spin` без `confidence: confirmed`.  
**Не добавляйте** в JSON-LD `sameAs` профили со статусом `candidate` или `ambiguous`.

## 5. Частые проблемы

### 5.1 Совпадение только по фамилии-инициалам

Инициалы не позволяют однозначно идентифицировать человека в eLIBRARY. Например, «А. М. Шустова» — в базе может быть несколько Шустовых А. М.  

**Решение:** Используйте `full_name_ru` если оно есть в очереди; сравнивайте список публикаций в РИНЦ с темами из архива.

### 5.2 Несоответствие транслитерации

eLIBRARY хранит Cyrillic имена, но OpenAlex — латинские. Транслитерация может быть нестандартной.  
Пример: «Рыжакова» → «Ryzhakova» или «Ryzhakova» или «Rjzhakova».

**Решение:** Проверяйте по Cyrillic-поиску в eLIBRARY и по Latin-поиску в OpenAlex независимо.

### 5.3 Советские учёные и неполное покрытие РИНЦ

Для учёных, активных до 1990-х, РИНЦ может не содержать ранних публикаций.  
Выборочное покрытие не означает, что человек не существует в базе.

**Решение:** Если найден профиль с частичным покрытием, но с явным совпадением организации — ставьте `candidate`, а не `confirmed`.

### 5.4 Один человек — два профиля в РИНЦ

Дубликаты в eLIBRARY не редкость. При обнаружении дубликатов:

1. Выберите профиль с более полным списком публикаций.
2. Зафиксируйте оба ID в поле `notes`: `"РИНЦ: два профиля: 123456 (основной), 654321 (дубликат)"`.
3. В `rinc_author_id` ставьте ID основного профиля.

## 6. Что НЕ делаем

| ❌ Запрещено | Причина |
|---|---|
| Автоматический скрейпинг eLIBRARY | Нарушение ToS, ненадёжность данных |
| Перенос `rinc_author_id` без даты `checked_at` | Невозможно отследить актуальность |
| Использование РИНЦ как source of truth для bio-данных | РИНЦ не верифицирует биографические записи |
| Добавление SPIN без проверки, что это правильный учёный | SPIN уникален, но профиль может быть у однофамильца |

## 7. Структура полей РИНЦ в authority_ids.json

```
rinc_author_id   — числовой ID автора в eLIBRARY (видно в URL профиля)
spin             — SPIN-код (8-значный идентификатор, формат XXXX-XXXX)
```

Пример URL профиля: `https://elibrary.ru/author_items.asp?authorid=123456`  
→ `rinc_author_id` = `"123456"`

Пример SPIN в профиле: `1234-5678`  
→ `spin` = `"1234-5678"`

---

## 8. Приоритеты проверки

Очередь `authority_review_queue.csv` отсортирована по `priority_rank`:

| priority_rank | Тип | Действие |
|---|---|---|
| 1 | Много докладов (≥ 5), нет ни одного ID | Первоочерёдная проверка |
| 2 | Активный учёный или имя с инициалами | Проверить при наличии времени |
| 3 | Нет preferred_latin_name | Сначала добавить Latin name, затем искать |
| 4 | Есть ID, но нет metadata (confidence, checked_at) | Заполнить метаданные существующих ID |
